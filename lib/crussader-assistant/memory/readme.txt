El sistema de memoria del asistente se basa en un modelo de estado conversacional persistido en la base de datos, concretamente en la columna assistant_memory de la tabla messaging_conversation. Esta memoria permite mantener el contexto entre mensajes y construir flujos conversacionales deterministas sin depender de memoria temporal en el servidor.

La estructura de memoria está dividida en dos secciones principales: profile y state. El bloque profile está reservado para información estable del usuario (preferencias, nombre, configuraciones, etc.), mientras que state contiene el estado activo de la conversación. Dentro de state se almacenan dos elementos fundamentales: pendingIntent y context.

pendingIntent representa la intención activa que el sistema está intentando completar. Contiene campos como requestedInstruction, action, product y subtype, que definen el tipo de operación solicitada por el usuario. Además incluye status, collectedData y missingFields, que permiten construir un flujo incremental de recopilación de datos. status puede tomar valores como WAITING_FOR_DATA, READY, COMPLETED o ABANDONED. collectedData almacena los datos ya extraídos del usuario (por ejemplo eventName, date, time) y missingFields mantiene la lista de datos que aún faltan para completar la acción.

El bloque context almacena información conversacional ligera que ayuda al asistente a formular la siguiente pregunta. Incluye lastAssistantQuestion, que permite recordar qué información se estaba solicitando al usuario, y lastUserMessages, que guarda un buffer corto de mensajes recientes del usuario (normalmente los últimos seis). Este buffer permite reconstruir parcialmente el contexto si el usuario responde de forma indirecta.

La memoria se gestiona a través del módulo assistantSessionMemory.ts, que contiene funciones para leer, actualizar y modificar el estado conversacional. Entre las funciones principales se encuentran getAssistantConversationState, que recupera el estado actual desde la base de datos, appendAssistantConversationUserMessage, que añade mensajes al buffer de contexto, setAssistantConversationLastQuestion, que guarda la última pregunta del asistente, y setAssistantConversationPendingIntent, que actualiza el pendingIntent.

El proceso de actualización de la intención se realiza mediante el módulo mergePendingIntentData.ts. Este módulo recibe el pendingIntent actual y el resultado del traductor de intención (TranslatorResult), fusiona los datos nuevos con los ya existentes y recalcula los campos faltantes (missingFields). De esta forma el asistente puede completar gradualmente la información necesaria para ejecutar una acción sin perder los datos ya proporcionados por el usuario.

Este diseño permite que el sistema mantenga conversaciones multi-turno de forma robusta y determinista, donde cada mensaje del usuario enriquece progresivamente el estado hasta que la acción queda completamente definida.
