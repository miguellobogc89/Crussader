# Manual Testing Flow - Google Calendar Integration

**Prerequisites**: 
- Staging environment running
- Google OAuth credentials configured (GOOGLE_CALENDAR_CLIENT_ID, CLIENT_SECRET)
- User logged in to Crussader
- companyId and locationId available

---

## 🔄 STEP 1: CONNECT GOOGLE

**Endpoint**: `GET /api/integrations/google/calendar/connect?companyId=ABC&locationId=XYZ`

**What happens**:
1. App redirects to Google OAuth consent screen
2. User grants permissions
3. Redirect to callback

**Expected outcome**:
- Redirect to Google OAuth login
- User sees permission dialog

---

## 🔄 STEP 2: OAUTH CALLBACK

**Endpoint**: `GET /api/integrations/google/calendar/callback?code=...&state=...`

**Database changes**:
```
ExternalConnection: CREATE/UPDATE
  - userId: <logged in user>
  - provider: "google-calendar"
  - access_token, refresh_token, expires_at: stored
  - status: "active"

external_calendar: CREATE/UPDATE for all Google calendars
  - For "Crussader Calendar": purpose = "crussader_mirror"
  - For all others: purpose = "google_context"
  - active: true
```

**Expected outcome**:
- User redirected to `/dashboard/calendar?connected=google_calendar`
- In DB: ExternalConnection exists
- In DB: external_calendar rows for all user's Google calendars

**Verify**:
```sql
SELECT id, accountEmail, status FROM "ExternalConnection" 
WHERE provider = 'google-calendar' AND userId = '<user-id>' LIMIT 1;

SELECT id, external_calendar_id, name, purpose, active 
FROM external_calendar 
WHERE provider = 'google-calendar' 
ORDER BY purpose DESC;
```

---

## 🔄 STEP 3: SYNC CALENDARS (Optional - Refresh List)

**Endpoint**: `POST /api/integrations/google/calendar/sync-calendars`

**Body**: None (uses authenticated user's connection)

**What happens**:
- Fetches latest calendar list from Google
- Updates/creates external_calendar records

**Expected outcome**:
- Response: `{ ok: true, created: X, updated: Y, total: Z }`
- DB: external_calendar updated with latest calendar properties (colors, names, etc.)

---

## 🔄 STEP 4: IMPORT GOOGLE EVENTS → CRUSSADER

**Endpoint**: `POST /api/integrations/google/calendar/sync`

**Body**:
```json
{
  "companyId": "ABC",
  "locationId": "XYZ"
}
```

**What happens**:
1. Finds all calendars with purpose = "google_context" for this location
2. Queries Google Calendar API for events (last 90 days)
3. Creates/updates external_calendar_event records
4. Creates Appointment records with externalProvider = "google-calendar"

**Expected outcome**:
- Response: `{ ok: true, imported: N, importedByCalendar: {...} }`
- DB: external_calendar_event rows created
- DB: Appointment rows created with:
  - externalProvider: "google-calendar"
  - externalCalendarId: <calendar_id>
  - externalEventId: <event_id>

**Verify**:
```sql
-- Check imported events
SELECT COUNT(*) FROM external_calendar_event 
WHERE provider = 'google-calendar' AND company_id = 'ABC';

-- Check imported appointments
SELECT COUNT(*) FROM "Appointment" 
WHERE externalProvider = 'google-calendar' 
AND locationId = 'XYZ';
```

---

## 🔄 STEP 5: EXPORT CRUSSADER APPOINTMENTS → GOOGLE

**Endpoint**: `POST /api/integrations/google/calendar/backfill`

**Body**:
```json
{
  "locationId": "XYZ"
}
```

**What happens**:
1. Finds crussader_mirror calendar for this location
2. Finds Appointments with no externalProvider (internal citas)
3. For each appointment:
   - Marks as "google-calendar-pending" (lock)
   - Creates event in Google crussader_mirror calendar
   - Updates Appointment.externalEventId
   - Sets externalProvider = "google-calendar"

**Expected outcome**:
- Response: `{ ok: true, total: N, exported: X, failed: Y }`
- DB: Appointment.externalEventId populated
- DB: Appointment.externalProvider = "google-calendar"
- Google Calendar: New events visible in "Crussader Calendar"

**Verify**:
```sql
-- Check exported appointments
SELECT COUNT(*) FROM "Appointment" 
WHERE locationId = 'XYZ' 
AND externalProvider = 'google-calendar'
AND externalEventId IS NOT NULL;
```

---

## 🔄 STEP 6: DISCONNECT GOOGLE

**Endpoint**: `DELETE /api/integrations/google/calendar/disconnect`

**Body**: None

**What happens**:
1. Finds user's ExternalConnection for google-calendar
2. Marks external_calendar.active = false for all google_context calendars
3. Cancels all Appointments with externalProvider = "google-calendar"
4. Sets ExternalConnection.status = "disconnected"

**Expected outcome**:
- Response: `{ ok: true, message: "google_calendar_disconnected" }`
- DB: external_calendar.active = false (for google_context)
- DB: Appointment.status = "CANCELLED" for imported events
- DB: ExternalConnection.status = "disconnected"
- IMPORTANT: Internal crussader appointments (no externalProvider) remain untouched

**Verify**:
```sql
-- Check disconnected state
SELECT id, status FROM "ExternalConnection" 
WHERE provider = 'google-calendar' AND userId = '<user-id>';

-- Check calendars marked inactive
SELECT COUNT(*) FROM external_calendar 
WHERE provider = 'google-calendar' 
AND purpose = 'google_context' 
AND active = false;

-- Check imported appointments cancelled
SELECT COUNT(*) FROM "Appointment" 
WHERE externalProvider = 'google-calendar' 
AND status = 'CANCELLED' 
AND locationId = 'XYZ';

-- CRITICAL: Check internal crussader appointments still exist
SELECT COUNT(*) FROM "Appointment" 
WHERE locationId = 'XYZ' 
AND (externalProvider IS NULL OR externalProvider = '');
```

---

## 🔄 VERIFY AFTER DISCONNECT

**CRITICAL CHECK**: Internal appointments must survive disconnect

```sql
-- Should return > 0 if you had internal crussader appointments
SELECT * FROM "Appointment" 
WHERE locationId = 'XYZ' 
AND (externalProvider IS NULL OR externalProvider = '')
LIMIT 5;
```

If this returns 0 rows, check that you created internal appointments BEFORE disconnecting.

## 🧪 TESTING CHECKLIST

- [ ] Step 1: Connect redirects to Google OAuth
- [ ] Step 2: Callback creates ExternalConnection + external_calendar rows
- [ ] Step 3: Sync-calendars updates calendar list
- [ ] Step 4: Sync imports Google events (count > 0)
- [ ] Step 5: Backfill exports internal appointments (count > 0)
- [ ] Step 6: Disconnect marks calendars inactive + cancels imported
- [ ] CRITICAL: Internal crussader appointments survive disconnect
- [ ] No duplicate imports if Step 4 is run twice
- [ ] No duplicate exports if Step 5 is run twice

---

## ⏱️ TIMING

Each operation should complete in seconds:
- Connect: ~1s (redirect)
- Callback: ~2s (API + DB writes)
- Sync-calendars: ~1s
- Sync (import): ~2-5s (depends on Google event count)
- Backfill (export): ~2-5s (depends on internal appointment count)
- Disconnect: ~1s

If any operation takes > 10s, check for:
- Google API rate limits
- Database indexing issues
- Large event/appointment counts
